04iloc vs loc — Row & Column SelectionMust Know
🧠
Memory Trick
iloc = integer position (end exclusive)  |  loc = label name (end inclusive)

iloc — Integer Position

python
df.iloc[1:, :]      # rows 1 to end → 999 rows
df.iloc[[0, 3]]     # rows 0 and 3 only
df.iloc[1:3]        # rows 1 and 2 (exclusive end)

# ❌ ERROR — iloc requires numeric indexers
# df.iloc[:, ['gender']]  → IndexError!

loc — Label-Based

python
df.loc[0]                         # single row
df.loc[[0, 3]]                    # rows 0 and 3
df.loc[0, ['gender']]             # row 0, gender col
df.loc[:, ['gender']]             # all rows, gender only → 1000 rows × 1 col
df.loc[10:, ['gender', 'lunch']] # rows 10+, two cols → 990 rows × 2 cols
df.loc[10:, 'gender':'lunch']    # col slice inclusive → 990 rows × 4 cols
05 — Filtering

Boolean Indexing & Filtering

Select only rows that match a condition — one of the most powerful Pandas skills.

05Single & Multiple ConditionsHigh Frequency
python
# Boolean mask
df['gender'] == 'female'   # True/False per row

# Apply mask
df[df['gender'] == 'female']          # 518 rows

# AND: both must be True

⚠️
Always Wrap Each Condition in ( )
& and | bind tighter than ==, > in Python. Without parentheses you get a TypeError every time.

A Series is a one-dimensional array with labels (index). Think of it as a single column from a spreadsheet. Every element has an index (0, 1, 2…) and a value.

14 — apply()

apply() — Custom Functions on Columns

Apply any Python function — built-in or custom — to every value in a column. The most flexible transformation tool in Pandas.

20apply() with Named and Lambda FunctionsHigh Frequency

Custom Function per Row Value

Define a function that takes a single value and returns the transformed result. Pass it to .apply() — Pandas calls your function once per row automatically.

python
# Step 1 — Define the function
def score_grade(val):
    if val <= 50:
        return 'Fail'
    return 'Pass'

# Step 2 — Apply it (called once per row)
df['score'] = df['avg'].apply(score_grade)

# Equivalent using a lambda
df['score'] = df['avg'].apply(lambda val: 'Fail' if val <= 50 else 'Pass')
▸ Before and after apply()
avg score 72.67 → Pass 82.33 → Pass 49.33 → Fail 76.33 → Pass

Real-World Example — Cleaning a Messy Column

apply() shines when you have multi-condition logic that can't be done with simple vectorized ops.

python
# Real data: Size column has '19M', '512k', 'Varies with device'
def clean_size(val):
    if 'M' in val:
        return val.removesuffix('M')      # '19M' → '19'
    if 'k' in val:
        return val.removesuffix('k')      # '512k' → '512'
    if val == 'Varies with device':
        return None

df['Size'] = df['Size'].apply(clean_size)
df['Size'] = df['Size'].astype(float)   # convert string → number
🧠
When to Use apply() vs Direct Operations
Use direct vectorized ops when possible (faster): df['col'] * 2, df['col'].str.upper()
Use apply() when you need conditionals (if/else), multiple conditions, or complex multi-step logic per value.
ApproachSpeedUse CaseExample
VectorizedFast ⚡Simple math/string opsdf['col'] + 10
apply(fn)ModerateComplex per-value logicdf['col'].apply(fn)

15 — DateTime

Working with Dates — pd.to_datetime() & .dt

Real-world data always has dates stored as strings. Learn to convert them and extract year, month, day for time-based analysis.

21pd.to_datetime() — Convert Strings to DatesTime Series

The Problem — Dates Loaded as Strings

When you load a CSV, date columns come in as object (string) type. You can't do date arithmetic or extract year/month without converting first.

python
# Load with encoding fix for special characters
df = pd.read_csv('Sample - Superstore.csv', encoding_errors='ignore')
# Alternative: encoding='latin1'

# Before: Order Date dtype is 'object' (string)
# Convert one column
df['Order Date'] = pd.to_datetime(df['Order Date'], format='mixed')

# Convert multiple date columns with a loop
for col in ['Order Date', 'Ship Date']:
    df[col] = pd.to_datetime(df[col], format='mixed')
▸ df.info() before and after
BEFORE: AFTER: Order Date object → Order Date datetime64[ns] Ship Date object → Ship Date datetime64[ns]

.dt Accessor — Extract Date Parts

Once a column is datetime64, the .dt accessor unlocks year, month, day, and more — all vectorized.

python
# Extract year → new column
df['year']  = df['Order Date'].dt.year
df['month'] = df['Order Date'].dt.month
df['day']   = df['Order Date'].dt.day

# Now group by year to see profit trends!
df.groupby('year').agg({'Profit': 'sum'})
▸ Yearly profit trend
Profit year 2014 49543.97 2015 61618.60 2016 81795.17 2017 93439.27

16 — String Methods

String Operations with .str

Pandas has a .str accessor that lets you apply Python string methods to entire columns at once — no loops needed.

22str.split(), str[0], str[-1] — Splitting TextString Ops

Splitting a Name Column into First/Last

The .str.split() method splits each string into a list. Chain .str[0] or .str[-1] to extract the first or last element from every list.

python
# Customer Name: 'Claire Gute', 'Darrin Van Huff', ...

# Extract first name
df['first_name'] = df['Customer Name'].str.split().str[0]

# Extract last name — str[-1] gets the last element
df['last_name'] = df['Customer Name'].str.split().str[-1]

# Access one row's split result (returns a plain Python list)
df['Customer Name'].str.split()[0]   # ['Claire', 'Gute']
▸ first_name column
0 Claire 1 Claire 2 Darrin 3 Sean ... 9990 Dave dtype: object
▸ last_name column
0 Gute 1 Gute 2 Huff 3 O'Donnell ... 9990 Brooks dtype: object

Other Useful .str Methods

python
df['col'].str.upper()              # 'hello' → 'HELLO'
df['col'].str.lower()              # 'HELLO' → 'hello'
df['col'].str.strip()              # remove leading/trailing spaces
df['col'].str.replace(',', '')     # '1,000' → '1000'
df['col'].str.removesuffix('M')   # '19M' → '19'
df['col'].str.contains('word')    # True/False — useful for filtering
🧠
The .str Chain Pattern
df['col'].str.split().str[0] — the first .str gives string methods on the Series; the second .str[0] indexes into the list created by split(). This double-.str chain is a common Pandas idiom.

17 — Exercises & Review

InterviewCommon Questions
loc vs iloc — what's the key difference?
loc uses labels (end inclusive), iloc uses integer positions (end exclusive). iloc only accepts numbers — passing a column name raises an IndexError.
What does axis=0 vs axis=1 mean?
axis=0 computes down each column (one result per column). axis=1 computes across each row (one result per row).
fillna() vs dropna() — when to use each?
Use fillna() when you want to impute missing values; use dropna() when rows with NaN are not useful. Dropping too many rows can skew analysis.
What is groupby()'s internal pattern?
Split-Apply-Combine: split rows into groups, apply a function to each group independently, combine results into a single output.
What does apply() do and when should you use it?
apply() calls a Python function once per row value. Use it for complex conditional logic (if/else) that can't be expressed as simple vectorized operations. For simple math or string ops, prefer direct vectorized methods which are faster.
Why convert a column with pd.to_datetime()?
Date columns load as strings (object dtype). After to_datetime(), they become datetime64[ns], which unlocks .dt.year, .dt.month, date arithmetic, and time-based groupby — none of which work on strings.
