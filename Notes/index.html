04iloc vs loc — Row & Column SelectionMust Know
🧠
Memory Trick
iloc = integer position (end exclusive)  |  loc = label name (end inclusive)

iloc — Integer Position

python
df.iloc[1:, :]      # rows 1 to end → 999 rows
df.iloc[[0, 3]]     # rows 0 and 3 only
df.iloc[1:3]        # rows 1 and 2 (exclusive end)

# ❌ ERROR — iloc requires numeric indexers
# df.iloc[:, ['gender']]  → IndexError!

loc — Label-Based

python

df.loc[[0, 3]]                    # rows 0 and 3
df.loc[0, ['gender']]             # row 0, gender col
df.loc[:, ['gender']]             # all rows, gender only → 1000 rows × 1 col
df.loc[10:, ['gender', 'lunch']] # rows 10+, two cols → 990 rows × 2 cols
df.loc[10:, 'gender':'lunch']    # col slice inclusive → 990 rows × 4 cols

05 — Filtering

Boolean Indexing & Filtering

Select only rows that match a condition — one of the most powerful Pandas skills.

05Single & Multiple ConditionsHigh Frequency
python
# Boolean mask
df['gender'] == 'female'   # True/False per row

# Apply mask
df[df['gender'] == 'female']          # 518 rows

# AND: both must be True
df[(df['gender'] == 'female') & (df['math score'] > 70)]  # 170 rows

# AND: male + standard lunch
df[(df['gender'] == 'male') & (df['lunch'] == 'standard')]   # 316 rows

# ❌ WRONG — TypeError without parentheses
# df[df['gender']=='female' & df['math score']>70]
⚠️
Always Wrap Each Condition in ( )
& and | bind tighter than ==, > in Python. Without parentheses you get a TypeError every time.

10fillna(), ffill(), bfill()Imputation

fillna() — Replace Missing Values

python
# Fill all NaN with a constant
df2 = df2.fillna(0)

# inplace=True → modifies original, don't assign result!
df2.fillna(0, inplace=True)   # ✅ correct

# Different strategy per column

10 — GroupBy

groupby() — Split, Apply, Combine

Group data by a category and compute statistics for each group.

14groupby() with get_group() and agg()Advanced
1
Split
Pandas splits rows into groups by column value (e.g. 'female': 518 rows, 'male': 482 rows).
2
Apply
An aggregation function (mean, sum, count…) is computed independently for each group.
3
Combine
Results are combined back into a single DataFrame indexed by the group values.
python
# Create GroupBy object
group = df.groupby('gender')

# Inspect group index → shows which rows belong to each group
group.groups   # {'female': [0,1,2,...], 'male': [3,4,...]}

# Extract a specific group as a DataFrame
female = group.get_group('female')  # 518 rows
male   = group.get_group('male')    # 482 rows

# Compute mean math score per gender
female['math score'].mean()   # 63.63
male['math score'].mean()     # 68.73

# Equivalent using agg on filtered df
female.agg({'math score': 'mean'})

# Most efficient: groupby + agg in one line
group.agg({'math score': 'mean'})
▸ group.agg({'math score': 'mean'})
math score gender female 63.633205 male 68.728216
💡
Real Insight from Your Data
Male students score ~5 points higher in math (68.7 vs 63.6). But don't stop there — female students typically outperform in reading and writing. Always compare all subjects before drawing conclusions!

⭐ agg() with Multiple Stats — The Key Pattern

Pass a list as the value in the dict to get multiple statistics for the same column in one shot. This is one of the most powerful and frequently tested patterns.

▸ Output

group.agg({

    'writing score': 'mean'
})

math score reading score writing score mean min mean max mean gender female 63.63 0 72.61 100 72.47 male 68.73 4 65.47 100 63.31

✏️GroupBy + agg() — Assignment QuestionsPractice

🎤 Interview Questions

What does groupby() return and how do you use it?
groupby() returns a DataFrameGroupBy object — nothing is computed yet. You chain .agg(), .mean(), or .get_group() on it. The internal pattern is: Split rows into groups → Apply a function → Combine results.
What is the difference between agg({'col': 'mean'}) and agg({'col': ['mean','min']})?
The first returns a single-level result with one stat. The second returns a MultiIndex column with both stats side by side — this is the list syntax that gives you multiple statistics per column in one call.
How do you group by more than one column?
Pass a list: df.groupby(['gender','lunch']). The result has a MultiIndex — one level per grouping column. This lets you do cross-segment analysis like "female + standard lunch" vs "male + free/reduced".
What aggregation functions can you pass to agg()?
'mean', 'min', 'max', 'sum', 'count', 'std', 'median', 'var', 'first', 'last', 'nunique'. You can also pass your own custom Python functions.

InterviewCommon Questions
loc vs iloc — what's the key difference?
loc uses labels (end inclusive), iloc uses integer positions (end exclusive). iloc only accepts numbers — passing a column name raises an IndexError.
What does axis=0 vs axis=1 mean?
axis=0 computes down each column (one result per column). axis=1 computes across each row (one result per row).
fillna() vs dropna() — when to use each?
Use fillna() when you want to impute (replace) missing values; use dropna() when rows with NaN are not useful. Dropping too many rows can skew analysis.
What is groupby()'s internal pattern?
Split-Apply-Combine: split rows into groups, apply a function to each group independently, combine results into a single output.
ffill vs bfill — what's the difference?
ffill propagates the last known value forward into NaN cells; bfill fills backward from the next known value. Both are used for time-series data.

python
import pandas as pd

# ── LOADING ──────────────────────────────────────────────
df = pd.read_csv("file.csv")

# ── INSPECTION ───────────────────────────────────────────
df.shape                     # (rows, cols)
df.head() / df.tail()         # preview
df.info()                    # types + null counts
df.dtypes / df.columns        # types / names
df.count()                   # non-null per column

# ── SELECTION ────────────────────────────────────────────
df['col']                    # Series (1D)
df[['c1','c2']]             # DataFrame (2D)
df.iloc[1:,:]               # position-based, exclusive end
df.loc[10:,'col1':'col2']  # label-based, inclusive end

# ── FILTERING ────────────────────────────────────────────
df[df['col'] == 'val']
df[(df['c1']=='v') & (df['c2']>70)]  # AND
df[(df['c1']=='v') | (df['c2']>70)]  # OR

# ── CATEGORICAL ──────────────────────────────────────────
df['col'].unique()            # distinct values
df['col'].nunique()           # count distinct (excl NaN)
df['col'].value_counts()      # frequency table

# ── AGGREGATION ──────────────────────────────────────────
df.mean(numeric_only=True, axis=0)  # per column
df.mean(numeric_only=True, axis=1)  # per row
df[['c1','c2']].agg(['mean','min','max'])
df.agg({'c1':'min', 'c2':'mean'})

# ── MISSING VALUES ───────────────────────────────────────
df.isnull().sum()             # count per column
df.isnull().sum().sum()      # total
df['col'].notna()             # True where not NaN
df.fillna(0)                  # fill with constant
df.fillna({'c': df['c'].mean()})  # fill per column
df.ffill()                    # forward fill
df.bfill()                    # backward fill

# ── CLEANING ─────────────────────────────────────────────
df.dropna()                   # drop rows with any NaN
df.dropna(how='all')          # drop rows all NaN
df.dropna(subset=['col'])     # drop if specific col NaN
df.drop(['c'], axis=1)       # drop column
df.drop(index=[0,1])          # drop rows
df.drop_duplicates()           # remove exact duplicate rows

# ── GROUPBY ──────────────────────────────────────────────
g = df.groupby('col')
g.get_group('val')            # extract one group
g.agg({'score': 'mean'})      # aggregate each group
